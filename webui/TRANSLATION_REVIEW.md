# 翻译审查指南

本文档提供了审查翻译质量的方法和建议。

## 审查方法

### 1. 对比审查法
将英文版本作为参考基准，逐一对比其他语言的翻译：

```bash
# 查看所有翻译文件的结构是否一致
cat webui/src/i18n/locales/en-US.json | jq 'keys'
cat webui/src/i18n/locales/fr-FR.json | jq 'keys'
# ... 其他语言
```

### 2. 术语一致性检查
确保专业术语在所有语言中保持一致性：
- "Funding Rate" → 资金费率 / Taux de financement / 資金調達レート
- "Position" → 持仓 / Position / ポジション
- "Order" → 订单 / Ordre / 注文

### 3. 上下文审查
翻译应该符合金融/交易领域的上下文：
- 避免直译，使用行业标准术语
- 确保文化适应性

### 4. 使用在线工具验证

#### 推荐的验证工具：
1. **DeepL** - 高质量的机器翻译参考
2. **Google Translate** - 快速对照检查
3. **专业术语词典**：
   - 金融术语词典
   - 加密货币术语表

### 5. 母语者审查
建议找各语言的母语者进行审查，特别是：
- 金融/交易领域的专业人士
- 熟悉加密货币术语的用户

## 需要特别注意的翻译

1. **Funding Rate** - 在某些语言中可能需要区分"资金费率"和"融资利率"
2. **K-Line/K线** - 中文术语，其他语言可能需要采用或翻译
3. **Strategy Slots** - 专业术语，需确保在目标语言中有合适的对应词

## 审查清单

对于每个翻译文件，请检查：
- [ ] 所有键都存在（与英文版本一致）
- [ ] 专业术语准确
- [ ] 语法正确
- [ ] 符合目标语言的文化习惯
- [ ] 长度适中（不会导致UI布局问题）
- [ ] 大小写符合目标语言的规范
