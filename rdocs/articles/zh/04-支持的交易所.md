## 选择建议

### 根据需求选择

#### 1. 追求最大流动性

**推荐：** Binance

**理由：**
- 交易量最大
- 流动性最好
- 支持主流币种

#### 2. 追求最低手续费

**推荐：** Phemex（零手续费）或 Gate.io（低手续费）

**理由：**
- Phemex 零手续费合约交易
- Gate.io 手续费率低
- 适合高频交易

#### 3. 需要批量操作

**推荐：** Bitget

**理由：**
- 原生支持批量下单/撤单
- 提高交易效率
- 减少 API 调用

#### 4. 需要测试功能

**推荐：** Binance、OKX、Bybit、MEXC 等（支持测试网）

**理由：**
- 可以在测试网验证策略
- 避免实盘风险
- 熟悉系统操作

#### 5. 中文用户

**推荐：** OKX、Huobi、Gate.io

**理由：**
- 中文支持好
- 客服响应快
- 文档完善

#### 6. 需要交易山寨币

**推荐：** MEXC、KuCoin、Gate.io

**理由：**
- 币种丰富
- 上新速度快
- 流动性好

### 综合建议

**首次使用：**
1. 选择支持测试网的交易所（如 Binance、OKX）
2. 在测试网验证策略
3. 确认无误后切换到实盘

**实盘使用：**
1. 选择流动性好的主流交易所（如 Binance、Bitget）
2. 根据手续费率选择（如 Phemex、Gate.io）
3. 根据需求选择功能（如批量操作选 Bitget）

## API 配置说明

### 通用配置

所有交易所都需要以下基本配置：

```yaml
exchanges:
  exchange_name:
    api_key: "YOUR_API_KEY"
    secret_key: "YOUR_API_SECRET"
    fee_rate: 0.0002  # 交易手续费率
```

### 特殊配置

#### 需要 Passphrase 的交易所

以下交易所需要额外的 `passphrase` 配置：

- Bitget
- OKX
- KuCoin

**配置示例：**

```yaml
bitget:
  api_key: "YOUR_API_KEY"
  secret_key: "YOUR_API_SECRET"
  passphrase: "YOUR_PASSPHRASE"  # 必需
  fee_rate: 0.0002
```

#### 支持测试网的交易所

以下交易所支持测试网：

- Binance
- OKX
- Bybit
- MEXC
- BingX
- Deribit
- BitMEX
- Phemex
- WOO X
- CoinEx
- Bitrue
- XT.COM
- BTCC
- AscendEX
- Poloniex
- Crypto.com

**配置示例：**

```yaml
binance:
  api_key: "YOUR_API_KEY"
  secret_key: "YOUR_API_SECRET"
  fee_rate: 0.0002
  testnet: true  # 使用测试网
```

### API 密钥权限设置

**建议权限：**
- ✅ 合约交易权限（必需）
- ✅ 查询权限（必需）
- ❌ 提币权限（不建议开启）
- ❌ 现货交易权限（如不需要）

**安全建议：**
- 设置 IP 白名单（如果支持）
- 定期更换 API 密钥
- 不要分享 API 密钥
- 使用只读 API 密钥进行测试

### 手续费率配置

**默认费率：**

不同交易所的默认手续费率不同，建议根据实际 VIP 等级设置：

```yaml
exchanges:
  binance:
    fee_rate: 0.0004  # Taker 费率（根据 VIP 等级调整）
  bitget:
    fee_rate: 0.0002  # 默认费率
  gate:
    fee_rate: 0.0002  # 默认费率
  phemex:
    fee_rate: 0.0000  # 零手续费
```

**注意：**
- 设置为 0 时，系统会使用交易所默认费率
- 实际费率取决于您的 VIP 等级
- 建议查看交易所官网费率表

## 交易所切换

### 切换步骤

1. **修改配置**

```yaml
app:
  current_exchange: "bitget"  # 切换到 Bitget
```

2. **配置新交易所**

```yaml
exchanges:
  bitget:
    api_key: "YOUR_BITGET_API_KEY"
    secret_key: "YOUR_BITGET_SECRET"
    passphrase: "YOUR_PASSPHRASE"
    fee_rate: 0.0002
```

3. **重启系统**

```bash
# 停止当前系统
Ctrl+C

# 重新启动
go run main.go
```

### 多交易所同时使用

系统支持同时使用多个交易所（多交易对配置）：

```yaml
trading:
  symbols:
    - exchange: "binance"
      symbol: "ETHUSDT"
      price_interval: 2
      order_quantity: 30
    - exchange: "bitget"
      symbol: "BTCUSDT"
      price_interval: 10
      order_quantity: 50
```

## 常见问题

### Q1: 如何选择交易所？

**A:** 根据需求选择：
- 追求流动性：Binance
- 追求低手续费：Phemex 或 Gate.io
- 需要批量操作：Bitget
- 需要测试：选择支持测试网的交易所

### Q2: 测试网和实盘有什么区别？

**A:**
- 测试网：使用虚拟资金，不会造成真实损失
- 实盘：使用真实资金，有真实盈亏

**建议：** 首次使用建议在测试网验证

### Q3: API 密钥权限如何设置？

**A:** 建议只开启：
- 合约交易权限
- 查询权限

不要开启：
- 提币权限
- 不必要的权限

### Q4: 如何降低手续费？

**A:**
1. 使用 Maker 订单（PostOnly），可能获得手续费返还
2. 提高交易所 VIP 等级
3. 选择手续费率低的交易所（如 Phemex）

### Q5: 支持哪些交易对？

**A:** 支持各交易所的永续合约交易对，建议选择：
- 主流币种（BTC、ETH 等）
- 流动性好的交易对
- 避免选择波动性过大的币种
